发表于 2024-07-31 更新于 2024-08-01 分类于 数据库

数据库的几大范式

  • 第一范式：原子性，如果数据库表中所有字段值都是不可分解的原子值，那么该数据库表满足了第一范式。
  • 第二范式：唯一性，第二范式需要确保数据库表中的每一列都和主键相关，而不能只与主键的某一部分相关
  • 第三范式：避免数据冗余，第三范式需要确保数据表中的每一列数据都和主键直接相关，而不能间接相关。

Mysql 结构

  • Mysql Server层：（存储过程、视图、触发器）binlog属于Server层的日志。主要分为：
    • 连接器：管理连接，权限认证
    • 查询缓存：sql命中则直接返回结果，mysql8.0版本已经去掉此功能
    • 分析器：分析sql，词法分析，语法分析（做什么）
    • 优化器：执行计划生成，索引选择（怎么做）
    • 执行器：操作引擎，执行sql，返回结果（做）
  • 存储引擎层：数据的存储与提取，提供读写接口，支持InnoDB、MyISAM、Memory等多个存储引擎。redolog、undolog是InnoDB的日志

JVM之类加载器

发表于 2018-07-26 更新于 2024-07-31 分类于 JVM

Java类加载器是Java运行时环境（Java Runtime Environment）的一部分，负责动态加载Java类到Java虚拟机的内存空间中。

类加载器

BootStrap ClassLoader（启动类加载器）

  • 负责加载/jre/lib目录下的jar包和Class文件。可以通过System.getProperty(“sun.boot.class.path”)获取加载的路径。

Extension ClassLoader（扩展类加载器）

  • 负责加载/jre/lib/ext或java.ext.dirs指定目录下的jar包。

App ClassLoader（系统类加载器或应用类加载器）

  • 负责加载ClassPath或java.class.path目录下的jar包和Class文件。可通过ClassLoader.getSystemClassLoader()来获取。
  • 可以通过指定java.system.class.loader来自定义系统类加载器。

传统架构问题

  • 系统资源浪费（水平扩展需要扩展整个应用，不能扩展单个模块）
  • 部署效率低（修改了一行代码，需要部署整个应用）
  • 技术选型单一

微服务特点

  • 微小度颗粒（细粒度）
  • 服务单一简单（责任单一性），易于开发、理解和维护
  • 运行隔离性，独立部署
  • 管理自动化
  • 扩展性高
  • 服务内部高内聚、低耦合

分布式理论

CAP

  • Consistency（一致性）

    所有节点同一时间的数据完全一致

  • Availability（可用性）

    每次请求都能获取到非错的响应

  • Partition Tolerance（分区容错）

    当某节点或网络分区故障的时候，系统仍能够对外提供满足一致性和可用性的服务
